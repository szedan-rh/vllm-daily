  - `docker/Dockerfile`
  - `docker/Dockerfile.rocm`
  - `docker/versions.json`

### Summary

**What changed and why**  
This PR enables sccache for Rust builds in CUDA and ROCm Dockerfiles. For CUDA, sccache installation is moved to the base stage so both csrc-build and rust-build share the same binary and remote cache configuration. For ROCm, environment variables are exported to let cargo discover sccache. The changes aim to improve build cache reliability across different machines and architectures, mitigating issues with BuildKit cache for Rust builds.

**Technical impact**  
Rust builds will now use sccache as the compiler wrapper, caching compilation artifacts remotely. This reduces build times for incremental builds and across CI runs. The sccache daemon runs on a separate port (4227) for Rust to avoid conflicts with the csrc-build sccache instance. The changes also add conditional logic to mount AWS credentials and show sccache stats before and after the build.

**Potential risks**  
If sccache misconfigures or fails, Rust builds may fail or produce incorrect artifacts. The sccache download URL is hardcoded to a specific version (0.8.1), which may become outdated. The AWS credentials secret mount is optional, but if sccache requires authentication and credentials are missing, caching may fail silently. The separate sccache port could conflict if not properly isolated.

**Key insights**  
The PR consolidates sccache setup into the base stage for CUDA, reducing duplication. It also reorders entries in versions.json without functional change. The use of conditional environment variables (e.g., SCCACHE_BUCKET=${USE_SCCACHE:+...}) ensures sccache is only configured when enabled. The PR addresses a known pain point with BuildKit cache reliability for Rust artifacts.
