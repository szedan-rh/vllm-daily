### Summary

**What changed and why**  
Added a new `kv_cache_dtype` field to `SpeculativeConfig` to allow the draft model's KV cache dtype to be set independently from the target model. This is needed because on Blackwell GPUs, the global `--kv-cache-dtype fp8` causes the drafter to fail due to lack of attention backend support for non-causal attention with FP8 KV cache. The change provides an escape hatch via `--speculative_config.kv_cache_dtype bfloat16`.

**Technical impact**  
Introduces a new configuration option that overrides the draft model's cache dtype when set. The change touches 6 files: the config class, argument parsing, and three spec decode implementations (dflash, dspark, eagle) plus the base proposer. In the proposer and eagle utils, the entire VllmConfig is replaced with a modified cache_config; in dflash and dspark utils, only the cache_config is replaced. This ensures the draft model uses the specified dtype while the target model retains its original setting.

**Potential risks**  
If the new field is set incorrectly (e.g., to an unsupported dtype), the draft model may fail to initialize or produce incorrect results. The change modifies VllmConfig replacement in multiple places, which could introduce subtle bugs if the replacement logic is not consistent. There is also a risk of performance regression if the draft model uses a different dtype than the target, as it may require additional format conversions.

**Key insights**  
The PR addresses a real hardware compatibility issue on Blackwell GPUs where FP8 KV cache is not supported for non-causal attention. The solution is minimal and non-invasive, adding only a single optional field. The manual test shows 95% accuracy on GSM8k, confirming the fix works. The change also includes a fix for argument parsing to convert hyphens to underscores in speculative_config keys, which is a necessary cleanup.

**Key insights**  
The bug was subtle: sleep mode zeroed the weight tensor (since it's not an nn.Parameter), and the batch-invariant path didn't check has_weight. The fix leverages the existing pass_weight flag and makes the kernel respect it. The PR includes a clear reproduction script and test plan.
