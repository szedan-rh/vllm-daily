### Summary

**What changed and why**  
Added a generic cuteDSL low-latency BF16 router GEMM (ll_bf16_gemm) that supports arbitrary (K, N) shapes, replacing the shape-locked dsv3_router_gemm. The PR includes two kernels: a dot-product kernel for M≤4 and a split-K kernel for larger M, both outperforming cuBLAS. The change also integrates the new kernel into the GateLinear layer's dispatch hierarchy and adds warmup and tests.

**Technical impact**  
Introduces a new kernel path in the MoE router that is competitive with the existing DSV3 specialized kernel on supported shapes and extends support to arbitrary dimensions (e.g., DeepSeek-V4-Flash with K=14400). The dispatch logic in GateLinear.forward now checks for cuteDSL availability first, then falls back to existing kernels. The kernel warmup is added to precompile common shapes. The test suite covers correctness, determinism, and edge cases for both dot-product and split-K paths.

**Potential risks**  
The new kernel requires SM90+ (Hopper/Blackwell) and the cuteDSL (CUTLASS Python) package; if unavailable, the fallback works but performance may degrade. The dot-product kernel serializes M, so latency scales linearly with M, which could be a regression for M>4 on small K (though split-K handles M>4 for K≥2048). The split-K kernel uses cluster-level DSMEM reduction, which may have subtle correctness issues on non-NVIDIA hardware or future architectures. The warmup adds startup time. The code is new and may contain bugs not caught by tests.

**Key insights**  
The split-K kernel achieves near-constant latency (~4us) regardless of M by partitioning K across 8 CTAs, solving the scaling problem of the dot-product approach. The dispatch heuristic uses a tuned table for specific (K,N) pairs to select the optimal dot-product max M. The PR demonstrates significant speedups over cuBLAS (up to 3.1x) and matches or exceeds the existing DSV3 kernel on its home shapes. The E2E serving results show modest TPOT improvements (0.1-1.6%) and better accuracy/latency in the benchmark.
