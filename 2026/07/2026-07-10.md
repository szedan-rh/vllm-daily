### Summary

Changed `torch.empty(permuted_kv_cache_shape)` to `torch.empty(permuted_kv_cache_shape, device='meta')` when computing strides for padded KV cache views. This avoids allocating a real tensor just to read its stride, reducing memory overhead.

**Technical impact**  
Reduces memory allocation by using a meta tensor instead of a real tensor for stride calculation. The behavior remains identical because stride values are the same for meta tensors.

**Key insights**  
Using meta tensors is a lightweight way to obtain shape and stride information without allocating memory. This pattern can be applied elsewhere to optimize memory usage.

The memory spike came from Python list holding all expert tensors simultaneously before torch.cat. Preallocating a single tensor and assigning per-expert avoids the spike. This pattern is common for memory-constrained environments and can be applied to similar repacking routines.

### Summary

**What changed and why**  
Fixes a bug where the thinking token budget was not enforced after a model naturally emits </think> and then re-enters a new <think> block. The root cause was that positional markers (start_thinking, end_thinking) and search scope were not reset on natural think-block exit, unlike the forced-end path fixed in #43757. The fix adds two mechanisms: (1) a scan_offset to scope searches past completed blocks, reset on any think-block exit, and (2) early natural-end detection before the countdown-based early return to handle adjacent </think> and <think> tokens without intervening content.

**Technical impact**  
Changes the thinking budget state machine in thinking_budget_state.py to properly handle multiple think blocks. Adds a scan_offset field to state entries, modifies _update_think_state to search from scan_offset instead of 0, and resets markers on both natural and forced exits. Also adds 12 new unit tests in test_correctness.py covering various re-entry scenarios. A minor refactor removes the _find_last_sequence_index_from helper and replaces it with slice-based search. The is_pin_memory parameter is threaded through to fix a test infrastructure issue.

**Potential risks**  
The changes to the state machine could introduce regressions in thinking budget enforcement for single-block cases or edge cases with continue_thinking. The early natural-end detection may interact with prompt continuation semantics. The removal of start_search_pos and end_search_pos fields could affect other parts of the codebase that depend on them (though they were only used internally). The test changes include a modification to an existing test (test_thinking_budget_long_thinking_section_end_marker_found_at_correct_index) that changes assertions, which could mask regressions if the new behavior is incorrect.

**Key insights**  
The bug was a classic state machine issue where exit paths were not symmetric: forced-end exit reset state but natural-end exit did not. The fix introduces a unified reset mechanism for both paths. The scan_offset approach is elegant because it avoids re-scanning already-processed tokens. The early natural-end detection is necessary because the countdown-based logic would never trigger when there is no content between blocks. The comprehensive test suite (12 tests) covers many edge cases and includes regression tests for existing behavior.

### Summary

**What changed and why**  
Removed preallocated output buffer from attention submodules in Qwen3NextDecoderLayer and QwenGDNLinearAttention, changing from slice assignment to returning tensors directly. This enables all-reduce RMSNorm fusion patterns that were previously blocked by the slice assignment, as shown by the increase from 0 to 2 replaced patterns in the fusion pass.

**Technical impact**  
Improves performance by allowing the all-reduce RMSNorm fusion optimizer to replace 2 patterns per layer instead of 0, reducing kernel launch overhead and memory bandwidth usage. The change also simplifies the interface by removing the output parameter and returning tensors directly.

**Potential risks**

**Key insights**  
Preallocated output buffers with slice assignment can prevent compiler fusion optimizations. Returning tensors directly allows the compiler to see the full computation graph and apply fusions like all-reduce + RMSNorm. This is a common pattern in performance optimization for transformer models.

  - `vllm/parser/harmony.py`

### Summary

**What changed and why**  
Added support for the `include_reasoning` parameter in non-Harmony models across the Responses API and unified Parser interface. Previously, `include_reasoning` was only supported for Harmony models and Chat Completions non-streaming. This change extends suppression of reasoning content to all endpoints, reducing network traffic and enabling safety-filtered deployments.

**Technical impact**  
Introduces a new field `include_reasoning` in `ResponsesRequest` protocol (default True). Modifies the abstract parser, harmony parser, and parser engine to suppress reasoning deltas in streaming when `include_reasoning=False`. Updates the Chat Completions serving logic to hide per-token metadata (logprobs, token_ids) when reasoning is suppressed. Adds 16 new tests and updates existing parser tests to set `include_reasoning=True` on mock requests.

**Potential risks**  
Backwards compatibility is preserved (default True). However, the change to suppress logprobs and token_ids when `include_reasoning=False` may break clients that rely on these fields even when reasoning is hidden. The modification in `_extract_tool_calls` to treat empty/whitespace content as no tool calls for required/named tool choices could alter behavior in edge cases. The new suppression logic in multiple parse_delta methods must be consistent to avoid double suppression or missed suppression.

**Key insights**  
The PR closes a feature gap by unifying `include_reasoning` behavior across all API endpoints and parser paths. The suppression is applied at the parser level rather than the serving layer, ensuring consistency. The addition of metadata suppression (logprobs, token_ids) when reasoning is hidden is a security measure to prevent leaking reasoning content through side channels.
