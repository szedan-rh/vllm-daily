  - `vllm/distributed/kv_transfer/kv_connector/v1/offloading/scheduler.py`
  - `vllm/v1/kv_offload/base.py`

### Summary

**What changed and why**  
Added a new configuration option `offload_prompt_only` to skip offloading decode-phase KV blocks to CPU. This is motivated by reasoning models that strip prior thinking across turns, making decode KV from completed turns unreusable. Offloading those blocks wastes PCIe bandwidth and evicts reusable prefill blocks from the CPU tier.

**Technical impact**  
Introduces a boolean flag `offload_prompt_only` in the offloading connector configuration. When enabled, the scheduler clamps the number of offloadable tokens to the prompt length, preventing decode blocks from being queued for store. This reduces GPU→CPU data transfer by up to 82% and improves CPU-tier hit rate by 6.1 percentage points, leading to 48% lower TTFT for late turns in multi-turn conversations.

**Potential risks**  
The flag defaults to `True` in `base.py`, which changes behavior for existing deployments that may rely on offloading decode blocks. Users who need decode offloading must explicitly set `offload_prompt_only=False`. The change only affects the offloading connector; other connectors are unaffected. Tests cover both synchronous and asynchronous scheduling modes.

**Key insights**  
The optimization leverages the observation that in multi-turn reasoning model conversations, decode KV from previous turns is often discarded due to thinking stripping. By skipping offload of these blocks, PCIe bandwidth is preserved for genuinely reusable prefill blocks, improving overall system efficiency. The implementation is minimal (4 files, +76/-0) and includes unit tests to verify the behavior.
