### Summary

**What changed and why**  
After AITER's shuffle_weights() and replace_parameter() calls in four MoE methods, the new nn.Parameter objects lost the custom is_shuffled attribute that the shuffle helper sets. Low risk. The change is minimal (adding attribute assignment after existing replace_parameter calls) and follows the same pattern already used in quark_moe.py. It only affects AITER backends and does not alter any other code paths. The attribute is only read by AITER's dispatcher, so no side effects on other backends.

**Key insights**  
The root cause is that replace_parameter creates a new Parameter object, which does not preserve custom attributes from the original tensor. This is a common pitfall when wrapping tensors in nn.Parameter. The fix is straightforward and aligns with existing practice in the codebase.
