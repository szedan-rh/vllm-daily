## 5. [[Perf] Avoid forward scan for async output placeholders](https://github.com/vllm-project/vllm/pull/42938)


### Base Information

- **PR Number:** #42938
- **Author:** [izikgo](https://github.com/izikgo)
- **Merged By:** [vllm-bot](https://github.com/vllm-bot)
- **Merged time:** 2026-05-19 20:16:08
- **Type:** `SUBSTANTIAL`
- **[Changed Files](https://github.com/vllm-project/vllm/pull/42938/files) (1):**
  - `vllm/v1/worker/gpu_input_batch.py`

### Summary

**What changed and why**  
Replaced a forward scan (list.index(-1)) with a backward scan to find the first -1 placeholder in async output token IDs. The forward scan was O(output_len) per request per decode step, causing significant CPU overhead for long generations. The backward scan is O(num_placeholders), which is typically small.

**Technical impact**  
Reduces per-step CPU time for patching async output placeholders from O(output_len) to O(num_placeholders). In benchmarks with 60k output tokens, end-to-end throughput improved from ~1.9k tok/s to ~12.2k tok/s, and mean TPOT dropped from 65.5ms to 9.9ms. The change is purely algorithmic and preserves existing behavior.

**Potential risks**  
Low risk. The backward scan assumes placeholders are always at the end of the list, which is true for current usage. If future changes insert placeholders elsewhere, the logic would break. The change is small and well-tested with syntax check and benchmark.

**Key insights**  
A simple algorithmic change from forward to backward scan eliminated a major CPU bottleneck in long-generation workloads with async scheduling and reasoning parsers. The performance improvement is dramatic (6x throughput) without any hardware changes.

### Summary

**What changed and why**  
Fixed FlashInferNVLinkOneSidedManager to grow the shared MNNVL workspace across heterogeneous MoE layers (e.g., quantized base + unquantized MTP head) instead of first-call-wins sizing. Previously, the workspace was sized from the first call and subsequent calls with larger payloads would overrun, causing an assert. The fix computes the needed workspace on each initialize() call, grows to the union, and rebuilds MoeAlltoAll only when necessary. **Technical impact**
