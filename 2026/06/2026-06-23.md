### Summary

Added support for GLM-5 model in the DSV3 router GEMM kernel by introducing a new hidden dimension (6144) alongside the existing 7168. **Technical impact**

**Key insights**  
The PR demonstrates a clean extension pattern for adding new model dimensions to a specialized CUDA kernel. The use of a set for unsupported shapes is a good defensive practice. The code refactoring in the dispatch logic (using a_ptr/b_ptr and nested if-else) improves readability. The test results indicate no regression in accuracy for the existing models, and the new GLM-5 support is functionally correct.

### Summary

**What changed and why**  
The condition for deferring prefills in the DP prefill cadence was changed from `request.num_computed_tokens == 0` to `num_computed_tokens < request.num_tokens - 1`. This fixes an over-throttling bug where requests with fully computed prompts (e.g., full prefix-cache hit or fully remote-KV-loaded decode) were incorrectly deferred, while requests resumed from preemption with output tokens to recompute could escape deferral. The new condition correctly defers only when there is local prefill work remaining.

**Technical impact**  
Improves scheduling accuracy for disaggregated prefill (DP) by ensuring that requests with no local prefill work (e.g., decode-only steps) are not unnecessarily delayed. This reduces latency for such requests and aligns deferral behavior with the intended DP cadence logic.

**Potential risks**  
Low risk. The change is a targeted fix to the deferral condition. However, edge cases where `num_computed_tokens` might be incorrectly tracked could lead to unexpected deferral or non-deferral. The tests added cover both fully and partially matched remote-KV scenarios, reducing risk.

**Key insights**  
The original condition `num_computed_tokens == 0` was too broad: it deferred requests that had no local prefill work (e.g., full prefix-cache hits) and failed to defer requests with partial local prefill (e.g., preemption resumes). The fix uses `num_computed_tokens < request.num_tokens - 1` to check if there is any remaining local prefill compute, which is the correct semantic for DP cadence deferral.
